es6面试题

js数据类型？

js基础数据类型：string、number、bool、Null、undefined、symbol(es6新增)
js复杂数据类型：Array , Object

var let const 区别？

let 具有块级作用 不能重复声明 可以重复赋值
const 具有块级作用域 不能重复声明 不能重复赋值
var 全局作用域 可以重复声明 可以重复赋值
暂时性死区：
    我个人理解，所谓的暂时性死区就是在块级作用域内使用let声明了变量，那么这个变量就不
    会受外部的影响，这个我把它理解为暂时性死区。

​ 还可以说是： 我们在let声明之前不能使用变量，这种现象被称为暂时性死区。

​ 变量提升：
​ 我个人理解，所谓的变量提升就是为了先事先声明变量，然后在进行赋值

数据存储方式？

js中基本数据类型存储在栈中存储的是值。
js中复杂的数据类型存储堆中存储的是地址，指针会指向这个内存地址。

深拷贝和浅拷贝？

1、只有复杂数据类型设计到了深、浅拷贝, 基本数据类型统称为变量赋值！
2、深拷贝是指拷贝对象的具体内容，二内存地址是自主分配的，拷贝结束之后俩个对象虽然存的值是一样的，但是内存地址不一样，俩个对象页互相不影响，互不干涉
3、浅拷贝是指对内存地址的复制，让目标对象指针和源对象指向同一片内存空间。注意：当内存销毁的时候，只想对象的指针，必须重新定义，才能够使用

原型？

每个函数、构造函数中都有一个 prototype的显示原型属性。属性值是函数的原型对象。原型对象中定义的属性与方法，都是被实例对象所调用的。一般原型对象指向的是 object类型的原型对象。
每一个实例对象身上有一个 隐式原型属性 __proto__属性值指向 函数的原型对象。

原型链: 实例对象和原型之间的连接 就称为原型链。

继承？

es5: 借调继承、组合继承、原型对象继承, 使用 call apply bind 函数改变this指向去实现
es6: 使用 extends 关键字继承 父类; constructor() 初识化属相、supper、 调用父构造器constructor()方法, 继承其属性

call、apply、bind函数

作用： 改变函数中的this指向！

  • 第一个参数 设置函数内部this的指向 ; 其它参数，对应函数的参数。

注意：call()与apply()、bind()区别在于：

  • 传递的参数不同 、 apply()的传参方式一个array形式;call()可以直接传递参数；
  • bind()函数 返回的是一个函数、需要加()调用

1.5 接收参数方式

rest-参数

  • ES6 引入 rest 参数（形式为...变量名），用于获取函数的多余参数（形参）。
  • 这样就不需要使用arguments对象了。
    • arguments（作用在函数体中，接受全部参数（数组类型））
  • rest 参数搭配的变量是一个数组，该变量将多余的参数放入数组中。

…rest 和arguments区别

  • arguments表示所有实参列表 不能使用在箭头函数中 只能出现在函数体的内部
  • ...rest 剩余参数 表示所有剩余参数
  • 当函数只有一个剩余参数，未传递其他参数此时 ...rest 和arguments是一致的 ，
  • ...rest只能出现在形参的最后面
    • function fn(…rest){ let arr = […rest] 操作数组}

注意：arguments 的值是伪数组， 数据集合； 可以使用 for 循环遍历可以使用 扩展运算符，转换真数组。

什么是深拷贝

只有复杂数据类型设计到了深、浅拷贝, 基本数据类型统称为变量赋值！

深拷贝是指拷贝对象的具体内容，二内存地址是自主分配的，拷贝结束之后俩个对象虽然存的值是一样的，但是内存地址不一样，俩个对象页互相不影响，互不干涉
